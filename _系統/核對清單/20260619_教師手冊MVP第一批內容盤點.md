# 20260619 教師手冊 MVP 第一批內容盤點

## 一、文件目的

本文件用於盤點教師手冊第一批最值得上架的內容，建立可被各處室共同理解的 MVP 內容清單、風險分級與上架順序。

本文件只做內容治理規劃，不新增後台資料、不修改前台或後台程式、不處理部署。

## 二、盤點原則

MVP 第一批內容以「老師每天或每週真的會用到」為優先，先放低風險、高頻、全體教師可查閱的入口與流程提醒。

盤點時採用以下原則：

* 先入口，後全文。
* 先低風險，後高風險。
* 先全體教師共用，再處理少數角色使用內容。
* 只放已公告、已確認、可公開給全體教師查閱的資料。
* 高風險內容只放「找誰、從哪裡進入、下一步提醒」，不放個案或細節。
* 每筆資料都應有主責處室與後續維護責任。

## 三、MVP 第一批內容範圍

MVP 第一批建議聚焦下列內容：

* 差勤 / 請假入口
* 校內行事曆
* 場地與設備借用
* 資訊帳號與常用系統
* 教師週報 / 公告入口
* 常用表單入口
* 新進教師第一週流程
* 校園空間與安全基本提醒
* 校安通報入口與「先找誰」流程提醒
* FAQ：帳號、請假、設備、場地、表單、Padlet

暫不把教師手冊做成完整行政百科。第一批目標是讓老師能更快找到常用入口、常見流程與承辦窗口。

## 五、風險等級定義

### 低

公開入口、一般表單、全體教師可用連結。

範例：

* 差勤系統入口
* 校內行事曆入口
* 常用表單索引
* 已公開的全校公告入口

### 中

流程說明、年度資料、需確認是否最新版。

範例：

* 場地借用流程
* 設備借用流程
* 新進教師第一週流程
* 年度行事曆或年度檔案入口

### 高

校安、性平、霸凌、特教、輔導、個資、採購工程、財產細節等，只能放入口或流程提醒，不放細節。

範例：

* 校安通報入口
* 性平或霸凌處理的承辦入口提醒
* 特教或輔導支持流程入口
* 採購、工程、財產相關的承辦窗口提醒

### 不可上架

學生個資、個案、成績、通報內容、未公告決策、含私人 token 或限時簽章連結。

此類資料不得放入教師手冊前台，也不應以附件、截圖、複製文字或外部連結形式繞過。

## 六、優先上架順序

1. 低風險常用入口：差勤、行事曆、週報、常用表單。
2. 高頻行政流程：場地設備、資訊帳號、新進教師流程。
3. 安全入口型內容：校安通報入口、校園空間與安全提醒。
4. FAQ 與 Padlet：依實際問題補齊。
5. 需處室確認資料：確認後再上架。

若同一批資料中有風險等級不同的項目，應先上架低風險資料，高風險資料等處室確認後再處理。

## 七、待處室確認項目

以下項目不應由維護者自行改寫或直接上架，需由主責處室確認：

* 校安、性平、霸凌處理入口與流程提醒
* 特教 / 輔導支持類資料
* 設備 / 場地借用最新流程
* 年度行事曆或年度文件
* 採購、財產、工程相關入口是否適合放入
* 各類表單是否為最新版本

確認方式建議記錄：

* 確認處室
* 確認角色或職稱
* 確認日期
* 是否可公開給全體教師
* 是否需要定期複查

## 八、不得直接上架項目

以下資料不得直接上架教師手冊：

* 學生個資
* 個案資料
* 成績資料
* 特教 / 輔導個案細節
* 校安通報細節
* 性平 / 霸凌個案內容
* 採購、工程、財產明細
* 未公告或未經處室確認的內部決策
* 帶私人 token、簽章、限時下載參數的連結
* 任何不適合全體教師共同查閱的資料

若資料是否可上架有疑義，預設不直接上架，先標記為待處室確認。

## 十、上架前檢核

每一筆 MVP 內容上架前，至少確認：

* 是否有主責處室
* 是否有承辦窗口或角色
* 是否為全體教師可查閱
* 是否不含個資 / 個案 / 成績 / 敏感內容
* 是否連結有效
* 是否使用安全 URL
* 是否符合目前學年度資料
* 是否已有更新日期
* 是否需處室確認
* 是否可回復或下架

若任一項無法確認，應先列為待整理或待處室確認，不直接上架。

## 十一、後續維護方式

建議採以下維護節奏：

* 每學期檢查常用入口與表單連結。
* 每學年檢查年度資料、新進教師流程與校務流程。
* 系統入口或表單異動時即時更新。
* FAQ 依實際問題滾動更新。
* 高風險資料每次重新上架前都需主責處室確認。
* 下架、重新上架與重大修改應保留稽核紀錄。

後續若要把本文件轉成實際資料匯入或後台新增作業，應另開施工 PR，不混入純文件 PR。

## 十二、正式結論

教師手冊 MVP 第一批內容應先從低風險、高頻、全體教師共用的入口與流程開始，逐步補齊待處室確認的中高風險內容。

本輪只建立內容盤點基準，不新增實際後台資料、不修改前台或後台功能、不部署 Worker 或 GAS。
