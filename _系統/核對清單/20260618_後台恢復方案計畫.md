# 20260618 後台恢復方案計畫

## 一、背景

* 教師手冊前台已恢復並完成 Phase 1 / Phase 2 優化。
* 後台仍維持維護頁。
* 後台恢復不應直接還原檔案，需先完成方案規劃、權限驗收與安全檢查。
* 本計畫只規劃後台恢復，不處理前台、Worker、GAS、LINE Bot / KV。

## 二、目前狀態

* `admin.html`：維護頁。
* `login.html`：維護頁。
* `prototype.html`：維護頁。
* 前台沒有「後台管理」入口。
* Worker / GAS 未 deploy。
* `main / origin/main` 同步。
* 工作區乾淨。

## 三、後台維護化來源

* 維護化 commit：`a033143 site: disable teacher handbook side entrances`
* 影響檔案：

  * `admin.html`
  * `login.html`
  * `prototype.html`
* 維護化前來源 tree：`a033143^`
* 後續如恢復，應優先參考：

  * `a033143^:admin.html`
  * `a033143^:login.html`
* 不建議使用備份檔直接覆蓋。
* 不建議恢復 `prototype.html`。

## 四、恢復範圍建議

### 方案 A：只恢復 `login.html` + `admin.html`

此為建議方案。

理由：

* 恢復最小可用後台登入與管理功能。
* 不重新開放 prototype 側門。
* 驗收範圍可控制在登入、角色權限與測試章節操作。

### 方案 B：恢復 `login.html` + `admin.html` + `prototype.html`

此方案不建議。

理由：

* `prototype.html` 屬側門或原型入口。
* 內容可能過時。
* 會增加驗收與安全風險。

### 方案 C：暫不恢復，只保留計畫

此方案適合目前階段。先完成本計畫文件，確認後台恢復邊界、權限驗收與安全檢查項目後，再另開測試分支施工。

## 五、權限驗收設計

### editor

* 可登入。
* 可儲存草稿。
* 可送審。
* 不可發布。
* 不可撤回。
* 不可刪除。

### reviewer / admin

* 可登入。
* 可查看待審資料。
* 可發布。
* 可撤回。
* 可刪除或回復測試資料。

驗收原則：

* 驗收必須使用測試章節或可回復測試資料。
* 不可用正式重要章節做破壞性測試。
* 後端 GAS 權限是最後防線，但 UI 仍需補驗。

## 六、前端安全檢查項目

恢復前必查：

* token 是否寫入 console。
* token 是否寫入 DOM。
* 是否有敏感 endpoint 外洩風險。
* 是否有 `innerHTML`。
* 是否有 `insertAdjacentHTML`。
* 是否有 `document.write`。
* 是否有 `javascript:` / `data:` URL 風險。
* 是否使用 `sessionStorage` 或 `localStorage`。
* 儲存內容是否只限短期登入狀態，不含敏感資料。
* 是否可能顯示學生個資、個案資料、採購工程細節等不應公開資料。

## 七、GAS 後端權限現況

* `publishChapter_()`：`requireReviewer_(user)`
* `saveDraft_()`：`requireEditor_(user)`
* `submitReview_()`：`requireEditor_(user)`
* `withdrawChapter_()`：`requireReviewer_(user)`
* `deleteChapter_()`：`requireReviewer_(user)`

補充說明：

* 後端發布權限已收斂為 reviewer / admin。
* 後台 UI 尚未驗收。
* 後續恢復後台時，不應同時修改 GAS 權限邏輯。
