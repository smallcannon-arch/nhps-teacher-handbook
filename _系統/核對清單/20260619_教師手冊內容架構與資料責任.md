# 20260619 教師手冊內容架構與資料責任

## 1. 文件目的

本文件用來定義「內湖國小教師手冊」的內容架構、資料責任、更新頻率與上架檢核原則。PR #9 僅新增文件，不新增功能、不調整後台、不修改前台、不部署 Worker 或 GAS。

本文件協助後續維護者判斷：

- 哪些資料適合放入教師手冊。
- 每筆資料應由哪個處室負責確認。
- 哪些資料不得放入教師手冊。
- 上架、下架與更新時應保留哪些稽核資訊。
- 第一批最值得優先整理的內容。

## 2. 目前教師手冊定位

教師手冊是給全體教師快速查找校務流程、常用入口、表單、提醒與承辦資訊的工作入口站。它不是正式公文系統、不是學生資料庫，也不取代各處室正式公告或雲端原始資料。

目前定位：

- 以全體教師可共同查閱的流程性、入口性、提醒性內容為主。
- 以最新學年度資料作為現行依據。
- 舊年度資料可作歷史參考，不作現行流程依據。
- 高風險內容需承辦處室確認後才可上架。
- 不放學生個資、個案、成績、通報細節或採購工程細節。

## 3. 教師手冊主分類

教師手冊主分類應維持穩定，避免因短期活動或個別文件而頻繁新增大分類。

建議主分類：

- 新進教師第一週
- 校園空間與安全
- 教務工作
- 學務與學生事件
- 輔導與特教支持
- 總務設備與場地
- 資訊系統與帳號
- 常用表單與 Padlet 入口
- 常見問題 FAQ
- 全校共用校務入口

分類使用原則：

- 一筆資料應優先歸入最主要的承辦處室或使用情境。
- 若資料跨處室，應指定主責處室，再於備註列出協作處室。
- 高風險資料即使有入口，也不應直接展開個案或細節內容。

## 4. 各處室資料責任

每筆資料至少應有以下責任欄位：

- 主責處室
- 承辦角色 / 職稱
- 資料類型
- 適用對象
- 更新頻率
- 最後確認日期
- 是否可公開給全體教師
- 是否含高風險內容
- 是否需處室確認
- 前台顯示狀態
- 備註

建議資料責任分工：

- 教務處：教務流程、課務、評量、學籍、教學相關系統與表單入口。
- 學務處：學生事件流程、校安入口、活動、導護、生活教育與安全提醒。
- 輔導室：輔導與特教支持流程、轉介入口、教師可查閱的一般提醒。
- 總務處：設備、場地、財產、修繕、校舍與總務支援流程。
- 資訊組 / 資訊承辦：資訊帳號、系統登入、設備借用或資訊安全提醒。
- 人事 / 會計：差勤、請假、薪資或經費相關入口與提醒。
- 全校共用：跨處室共用入口、常用表單、年度校務入口與 Padlet。

責任原則：

- 內容上架前應有主責處室。
- 高風險內容不得由非承辦者自行改寫為正式流程。
- 若資料來源與現行流程不一致，以最新學年度資料與承辦處室確認為準。

## 5. 文件類型與更新頻率

建議依資料類型設定最低檢查頻率：

- 常用入口：每學期檢查。
- 表單連結：每學期檢查，異動即更新。
- 流程說明：每學年檢查。
- 行事曆 / 年度資料：每學年更新。
- 校安 / 性平 / 霸凌 / 學生事件：依最新法規與處室確認，不直接自行改寫。
- 設備 / 場地借用：每學期或系統異動時更新。
- FAQ：依實際問題滾動更新。

更新頻率原則：

- 若連結、表單或承辦窗口已失效，應立即下架或標記待整理。
- 若資料涉及法規、校內流程或處室權責，更新前應由承辦處室確認。
- 年度資料應標明適用學年度，避免教師誤用舊資料。

## 6. 不得放入教師手冊的資料類型

教師手冊不得放入下列資料：

- 學生個資。
- 個案資料。
- 成績資料。
- 特教 / 輔導個案細節。
- 校安通報細節。
- 性平 / 霸凌個案內容。
- 採購、工程、財產細節。
- 未公告或未經處室確認的內部決策。
- 帶有私人 token、簽章、限時下載參數的連結。
- 任何不適合全體教師共同查閱的資料。

可放入的替代內容：

- 一般流程提醒。
- 「先找誰」的承辦窗口。
- 正式入口或表單連結。
- 不含個資與個案細節的注意事項。
- 需處室確認的提醒文字。

## 7. 上架前檢核清單

每筆資料上架前，應至少確認：

- 是否有主責處室。
- 是否有承辦窗口或角色。
- 是否為全體教師可查閱。
- 是否不含個資 / 個案 / 成績 / 敏感內容。
- 是否連結有效。
- 是否使用安全 URL。
- 是否符合目前學年度資料。
- 是否已有更新日期。
- 是否需處室確認。
- 是否可回復或下架。

建議補充檢查：

- 標題是否讓教師能快速理解用途。
- 說明是否避免放入內部未確認結論。
- 若資料屬高風險，是否已標記需處室確認。
- 若連結指向外部系統，是否仍需帳號權限。

## 8. 下架與更新原則

下架與更新應遵守：

- 資料過期即下架或標記待整理。
- 連結失效先下架，再補正。
- 高風險資料需承辦處室確認後才可重新上架。
- 年度資料以最新學年度為主。
- 舊年度資料可作歷史參考，不作現行流程依據。
- 下架與重大修改需保留稽核紀錄。

下架後處理方式：

- 可修正者：更新後重新上架。
- 需確認者：標記待整理，請承辦處室確認。
- 不適合公開者：維持下架，不放入前台。
- 涉及敏感資料者：不得轉貼到教師手冊內容欄位。

## 9. MVP 第一批建議內容

第一批最值得優先放入教師手冊的內容：

- 差勤 / 請假入口。
- 校內行事曆。
- 場地與設備借用。
- 資訊帳號與常用系統。
- 教師週報 / 公告入口。
- 常用表單入口。
- 新進教師第一週流程。
- 校園空間與安全基本提醒。
- 校安通報入口與「先找誰」流程提醒，不放個案內容。
- FAQ：帳號、請假、設備、場地、表單、Padlet。

MVP 原則：

- 先放高頻、低風險、全體教師都會用到的內容。
- 先放入口與提醒，再逐步補正式流程。
- 高風險流程只放承辦窗口、通報入口與一般提醒，不放個案細節。
- 不追求一次完整，優先讓教師可以找到正確方向。

## 10. 資料治理與稽核方式

建議資料治理方式：

- 每筆資料保留主責處室與最後確認日期。
- 高風險資料上架前保留處室確認紀錄。
- 下架、重新上架、重大修改需保留稽核紀錄。
- 測試資料應使用明確測試前綴，並於測後下架、停用或清理。
- 臨時測試帳號應於驗收後停用，並保留資料列作為稽核痕跡。

稽核重點：

- 前台顯示內容是否符合全體教師可查閱原則。
- 是否有不應公開的學生、個案、成績或採購工程資料。
- 是否有失效連結或帶私人簽章的連結。
- 是否有未經處室確認的高風險內容。
- 是否有資料長期待整理而未處理。

## 11. 後續擴充原則

後續擴充教師手冊內容時，應遵守：

- 每次新增高風險分類或高風險內容，需先確認資料責任。
- 不因單一文件或短期活動任意新增主分類。
- 不以教師手冊取代正式公告、公文、雲端原始資料或承辦處室說明。
- 不在教師手冊中保存不適合全體教師查閱的資料。
- 不把尚未確認的內容寫成正式流程。
- 若要新增欄位、調整後台流程或改變前台呈現，應另開 PR。
- 若要改變角色權限或資料上架 gate，需重新 QA。

結論：

- 教師手冊應先成為穩定、可查、低風險的校務入口。
- 資料責任應明確到處室與承辦角色。
- 高風險內容應保守處理，以入口、提醒與窗口為主。
- 後續內容擴充應分階段、可稽核、可回復。
